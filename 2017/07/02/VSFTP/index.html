VSFTP理论

VSFTP 搭建

本机环境： CentOS

RPM包:yum -y install vsftpd

主配置文件:vim /etc/vsftpd/vsftpd.conf

配置文件讲解

  • 匿名权限控制

anonymous_enable=YES:启用匿名访问

anon_umask=022：匿名用户所上传文件的权限掩码

anon_root=/var/ftp:匿名用户的FTP根目录

anon_upload_enable=YES：允许上传文件

anon_mkdir_write_enable=YES:允许创建目录

anon_other_write_enable=YES:开放其他写入权

anon_max_rate=0:限制最大传输速率（字节/秒

  • 本地用户权限控制

local_enable=YES:是否启用本地系统用户

local_umask=022:本地用户所上传文件的权限掩码

local_root=/var/ftp:设置本地用户的FTP根目录

chroot_local_user=YES:是否将用户禁锢在主目录

local_max_rate=0:限制最大传输速率（字节/秒）

  • 常用的全局配置项

listen_address= IP :设置监听的IP地址

listen_port=21:设置监听FTP服务的端口号

write_enable=YES:是否启用写入权限

download_enable=YES:是否允许下载文件

max_clients=0:限制并发客户端连接数

max_per_ip=0:限制同一个IP地址的并发连接数

pasv_enable=YES #启用被动模式

pasv_min_port=5000 :将客户端的数据连接端口改在50000-60000之间

pasv_max_port:60000

  • 限制访问

userlist_enable=YES:是否启用user_list列表文件

userlist_deny=YES:禁止列表文件里面的用户访问

userlist_deny=NO:允许列表里面的用户访问

  • 其它安全选项

connect_timeout=60 :连接超时时间

accept_timeout=60：请求超时时间

data_connection_timeout=300 ：数据连接超时时间

idle_session_timeout=300 ：会话超时时间

ftpd_banner=xxx和banner_file=/path/file ：显示登陆信息

tcp_wrappers=YES 启用tcp_wrappers，即/etc/hosts.allow和/etc/hosts.deny

VSFPT 实验：匿名用户

配置主配置文件:vim /etc/vsftpd/vsftpd.conf

先备份下配置文件：cp -a /etc/vsftpd/vsftpd.conf /etc/vsftpd/vsftpd.conf~


启动服务器：systemctl  start vsftpd
  • 测试：客服端安装：yum -y install ftp



一般匿名用户不会使用

VSFTP 实验：本地用户

配置主配置文件:vim /etc/vsftpd/vsftpd.conf


创建本地用户：useradd test 密码：123

在VSFPT服务器配置文件里添加：
chroot_local_user=YES #是否将用户禁锢在根目录，务必要加上这条
local_root=/var/rootftp #设置ftp的根目录
重启服务，客户端进行测试。

VSFTP 实验：虚拟用户

建立虚拟FTP用户的账号数据库文件

  • cd /etc/vsftpd/

    • [root@Centos-7 vsftpd]# vim vsuser

  • [root@myrhel1 vsftpd]# db_load -T -t hash -f vsuser vsuser.db

    • #-T：生成数据文件 -t：指定算法为hash -f：指定账号文件
    • #修改文件权限，不让任何人查看 chmod 600 vuser.

  • 创建FTP根目录及虚拟用户映射的系统用户

    • useradd -d /var/ftps/ -s /sbin/nologin vsuser

  • 建立支持虚拟用户的PAM认证文件，添加虚拟用户支持

    • vim /etc/pam.d/vsftpd

      #%PAM-1.0
      auth required pam_userdb.so db=/etc/vsftpd/vsuser
      account required pam_userdb.so db=/etc/vsftpd/vsuser

  • 在vsftpd.conf文件中添加支持配置,

  • 为个别虚拟用户建立独立的配置文件，启动服务并测试

    • mkdir /etc/vsftpd/user

  • vim /etc/vsftpd/user/user1 #文件名必须和虚拟用户名一致

    • 设置权限
    • anon_upload_enable=YES 允许上传
    • anon_mkdir_write_enable=YES 允许用户创建目录
    • anon_other_write_enable=YES 允许其它写入权限
    • 重启服务：systemctl restart vsftpd 测试

    注意：配置文件不能有空格

VSFTP 实验：加密

ftp是明文传输，极不安全，可以使用tcpdump等抓包工具进行抓取

  • 安装：yum -y install openssl openssl-devel 安装加密软件

  • openssl req -new -x509 -nodes -out vsftpd.pem -keyout vsftpd.pem

    • req 表示生成证书
    • -new表示生成一个新的证书
    • -x509证书格式
    • -nodes表示生成证书时，不生成密码保护，而默认会生成密码保护
    • -out指定输出证书的文件名
    • -keyout生成私钥的文件名
    • 根据提示填写好相关信息即可生成一个证书
    • chmod 400 vsftpd.pem #证书只允许管理员只读

  • 拷贝证书到ssl的默认证书目录:cp vsftpd.pem /etc/ssl/certs/

  • 配置VSFTP配置文件：vim /etc/vsftpd/vsftpd.conf

    • 添加：

ssl_enable=YES #表示启用ssl功能
allow_anon_ssl=YES #表示允许匿名用户使用ssl加密
force_local_data_ssl=YES #强制本地用户数据传输使用ssl加密
force_local_logins_ssl=YES #强制本地用户登录时使用ssl加密
force_anon_data_ssl=YES #强制匿名用户数据传输使用ssl加密
force_anon_logins_ssl=YES #强制匿名用户登录时使用ssl加密
ssl_tlsv1=YES #加密使用第一版协议
ssl_sslv2=NO #必须声明不使用ssl版本2，这个不能不加
ssl_sslv3=NO #必须声明不使用ssl版本3，这个不能不加
rsa_cert_file=/etc/ssl/certs/vsftpd.pem #指定生成证书的位置
service vsftpd start
最后使用专门的客户端工具（支持加密访问的客户端，命令行没有），如flashFXP

VSFTP 实验： 端口限制

pasv_min_port=10221 限制最小端口
