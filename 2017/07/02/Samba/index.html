Samba理论+配置文件

Samba 搭建

本机环境：CentOS

RPM包：yum -y install samba

主配置文件：vim /etc/samba/smb.conf

配置文件检查工具：testparm

Samba 实验：匿名用户登陆

备份配置文件：cp -a /etc/samba/smb.conf  /etc/samba/smb.conf~

配置文件：vim /etc/samba/smb.conf
  • security:安全级别设置
  • 开启windows 工作组，可以在windos登陆
  • 开启共享目录
  • 客户端：smbclient -L //IP/共享目录名
  • 客户端：smbclient //1.1.1.40/ceshi 不能用共享目录登陆，必须用共享名登陆

Samba 实验：用户登陆

配置文件：vim /etc/samba/smb.conf


  • useradd zz #添加本地用户

    • 添加Samba用户

      处理命令 pdbedit -a 系统存在用户
      查看已处理用户命令 pdbedit -L
      删除已处理用户命令 pdbedit -x

  • smbclient -U zz //1.1.1.40/ceshi 登陆 -U 指定用户登陆


Samba 实验：权限设置

配置文件:vim /etc/samba/smb.conf




指定权限分割





  • [test]

    comment = test 说明
    path = /test 共享目录名称
    public = yes 是否允许所有人访问
    read only = yes 只读
    write list = zhangsan 指定某用户只读
    valid users = lisi 指定合法用户或组，只有写在这里的才可以登陆
    create mask = 0755 创建文件掩码值
    directory mask = 0755 创建目录掩码值

  • 正常授权都使用：setfacl 来设置权限。在samba里设置权限不安全。

Samba 实验： 别名-限制

限制
  • 修改配置文件：yum -y install samba
    • 在主配置文件的 global 区域内声明
    • hosts allow = 192.168.1. 申明允许网段


  • hosts allow = 192.168.1.1 申明允许 IP
  • hosts deny = 192.168.1. 申明拒绝网段
  • hosts deny = 192.168.1.1 申明拒绝 IP

说明：允许 allow 与拒绝 deny 不能同时出现，只能单独出现。出现 allow 代表只允许出现 IP / 网段访问，其它全部拒绝。出现 deny 代表只拒绝出现 IP / 网段访问，其它全部允许。

别名

启用别名文件：在主配置文件的 global 区域内声明 username map = /etc/samba/smbusers

  • 配置文件：vim /etc/samba/smbusers

Samba 实验：图形化

安装 samba-swat samba 的图形化配置

  • 修改 samba-swat 的配置文件:vim /etc/xinetd.d/swat

将“only_from=127.0.0.1”改成“only_from=0.0.0.0”，这样该服务器就可以监听所有的来访IP
将“disable=yes”改成“disable=no”，这样swat子进程就可以随xinetd超级守护进程一起启动了。可以吃用chkconfig命令查看swat是否启用了。
swat默认使用tcp的901端口。“port=901”，可以修改
通过web来配置samba，默认使用root账号进入。“user=root”，可以修改成其他的系统用户
swat的执行程序默认在/usr/sbin目录下

启动服务：service xinetd restart

查看端口是否开启 netstat -an | grep 901
